2. How it should behave

QuestionProposed answer
When does it show?On the first page load of the calendar day, per browser. We store the last-shown date; if it isn't today, show the popup once.
"Try it now"Redirects the browser to https://act.38degrees.org.uk/admin/#media-library/items and records today as seen.
"Maybe later" / close (×)Closes the popup and records today as seen, so it won't reappear until tomorrow.
Where is "seen today" stored?A small cookie / localStorage key holding the date, e.g. flow_announce_seen=2026-06-22. No database or server change needed.
Which users?Everyone using the app — it lives in the shared layout. (Optionally only the 38degrees theme.)
Dismissible?Yes — close button, "Maybe later", clicking the dark backdrop, or pressing Esc all dismiss it.
"First time today" is intentionally per-day, not once-ever — it acts as a gentle, repeating nudge while people get used to Flow, without nagging on every page load.

3. Implementation plan

Mah Bucket is a Rails 6.1 app using theme-based layout partials (app/views/layouts/themes/<theme>/) with jQuery + Turbolinks already loaded. The popup is purely front-end, so it slots in cleanly with no controller, model or migration changes.

1

ERB Add a popup partial

Create app/views/layouts/themes/38degrees/_flow_announcement.html.erb containing the modal markup (the dialog you see in the mockup above), rendered hidden by default.

2

ERB Render it from the layout

Include it once in app/views/layouts/application.html.erb, just inside <body>, so it's available on every page:

<!-- app/views/layouts/application.html.erb -->
<body>
  <%= render "layouts/themes/#{theme}/header" %>
  <%= render "layouts/themes/#{theme}/flow_announcement" %>
  <%= yield %>
  ...

A default-theme version can be an empty partial so non-38-Degrees deployments show nothing.

3

JS Add the show-once-a-day logic

Add a small script (e.g. app/assets/javascripts/flow_announcement.coffee or plain JS) that runs on load, checks the date marker, and shows the popup when needed. Outline:

// runs on turbolinks:load
var KEY = 'flow_announce_seen';
var today = new Date().toISOString().slice(0,10); // 2026-06-22
var seen = localStorage.getItem(KEY);

function markSeen(){ localStorage.setItem(KEY, today); }

if (seen !== today) {
  showModal();                          // reveal the partial
}

// Try it now
onClick('#flow-now', function(){
  markSeen();
  window.location.href =
    'https://act.38degrees.org.uk/admin/#media-library/items';
});

// Maybe later / close / backdrop / Esc
onClick('#flow-later, .flow-close, .flow-backdrop', function(){
  markSeen();
  hideModal();
});
4

CSS Add styling

Add app/assets/stylesheets/flow_announcement.scss (auto-included via require_tree) for the overlay, modal card, brand-orange header and buttons — matching the styling in the mockup above.

5

Tidy-ups & tests

  • Make the URL and copy easy to change (a constant / data- attribute).
  • Optional RSpec/system test asserting the partial renders and the link points at the right URL.
  • Accessibility: role="dialog", focus trap, Esc to close (already in the mockup markup).

4. A few things to confirm

Frequency

Once per day per browser (recommended), or once ever until dismissed permanently? The mockup assumes per-day.

Audience

All themes, or only the live 38degrees theme? The plan scopes it to 38 Degrees by default.

Copy & tone

Happy with the wording in the mockup, or want it tweaked (headline, button labels)?

Storage

localStorage (survives, simple) vs a cookie (also sent to server). Both are front-end-only; recommending localStorage.
